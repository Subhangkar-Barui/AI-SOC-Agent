You are an expert full-stack developer, cybersecurity engineer, SOC analyst, application security engineer, and technical mentor.

Build a complete professional resume-ready project named:

AI-Powered SOC Dashboard with Downloadable Windows Network Monitoring Agent

Main project objective:
Create a full-stack cybersecurity monitoring platform where a user visits a React web dashboard, registers/logs in, downloads a Windows Network Monitoring Agent, installs it on their own local Windows system, pairs it with their dashboard account, and then monitors live network traffic metadata from their own home/personal/lab/authorized network. The agent sends traffic logs to the backend, the backend stores and analyzes them, and the React frontend displays live reports, alerts, devices, risk scores, and dashboard analytics for that specific user.

Important architecture rule:
The Windows Agent must not send data directly to the React frontend. The correct architecture is:

Windows Agent
→ FastAPI Backend API
→ MongoDB Atlas Database
→ React Frontend Dashboard

Reason:
React is only the visual interface. The backend must handle authentication, storage, analysis, alert generation, user-specific access control, and API security.

Use free or student-friendly resources:

* GitHub for source control
* Render Free Web Service for FastAPI backend
* MongoDB Atlas Free cluster for database
* Vercel Hobby for React frontend
* GitHub Actions for optional CI/CD
* Free Python libraries for Windows Agent
* Free frontend libraries

Final high-level workflow:

1. User visits the React website.
2. User registers or logs in.
3. User opens the Windows Agent page.
4. User clicks Download Agent.
5. User generates an Agent Pairing Key from the dashboard.
6. User installs/runs the Windows Agent locally.
7. Agent asks for backend URL and pairing key.
8. Agent registers itself with the backend.
9. Agent starts heartbeat updates.
10. Agent discovers local network devices.
11. Agent captures safe traffic metadata only.
12. Agent sends metadata to FastAPI backend.
13. Backend validates, stores, and analyzes the data.
14. Backend generates alerts for suspicious traffic.
15. React dashboard shows live traffic, devices, alerts, analytics, and reports.
16. User can close alerts, filter traffic, upload CSV logs, and view security reports.

Ethical and legal boundary:
The Windows Agent must only be used on the user’s own device, personal/home/lab network, or a network where the user has explicit permission. The project must not include stealth behavior, persistence abuse, bypassing detection, password capture, cookie capture, private message capture, or packet payload spying. The agent should collect metadata only, not sensitive payload content.

9. Detection logic:
   Create detection.py.

Implement rule-based detection first.

Rules:

* High severity log creates alert.
* New unknown device discovered creates Low/Medium alert.
* Same source IP connecting to more than 10 unique destination ports in 60 seconds creates Possible Port Scan alert.
* Repeated connection attempts to sensitive ports creates alert.
* Sensitive ports:

  * 22 SSH
  * 23 Telnet
  * 445 SMB
  * 3389 RDP
  * 3306 MySQL
  * 5432 PostgreSQL
* Unusual destination port creates suspicious status.
* Very high packet rate creates suspicious status.
* Large outbound transfer creates suspicious status.
* Traffic to custom blocked IP list creates High alert.

Risk score:

* 0–30 Normal
* 31–60 Suspicious
* 61–100 High Risk
